# iso9660

<img src="../../assets/fs/go-filesystems-iso9660.png" width="110" align="right">

ISO 9660 / ECMA-119 optical-disc filesystem (`.iso` images).

Implements [`filesystem.Filesystem`](../interface.md). No cgo, no root.

## Status

| Read | Write | Format | Label | Symlinks | On-disk format |
|:--:|:--:|:--:|:--:|:--:|---|
| ✅ | ✕ | ✕ | — | ✕ | ISO 9660 / ECMA-119 base; Rock Ridge + Joliet planned |

## Supported

- Primary Volume Descriptor scan (`CD001`), logical block size, root record
- Directory-record parsing + per-sector padding-aware directory walk
- Contiguous single-extent file reads (multi-sector)
- Case-insensitive path resolution (base ISO names are uppercased; `;version` stripped)

## Not implemented

- Write/format — ISO 9660 is a read-only format; mutators return `ErrReadOnly`
- Rock Ridge (POSIX names/perms/symlinks) and Joliet (UCS-2 long names)
- Multi-extent files
- Symlinks (`ReadLink` → `ErrNotSymlink`; requires Rock Ridge)

go get github.com/go-filesystems/iso9660
